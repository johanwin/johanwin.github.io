Skills

My goal has always been to obtain knowledge about the latest web technology to be more relevant. The following items briefly describes my main strengths.

HTML

I have mastered the inputting of angle brackets into text editors. Since the introduction of Netscape's <blink> tag, I have been typing HTML tags in a clean and accurate manner. Currently, I have used the HTML5 semantic elements to properly manage the structure of a webpage. In addition, I use Sublime Text text editor's HTML Tidy extension quite regularly to make my markup code easily readable for other developers, designers, and project managers.

Tools: HTML5 and HTML Tidy

CSS

I religiously checked CanIUse.com to determine which web browsers could render certain CSS3 and HTML5 code. The Sass CSS pre-processor is used to manage my CSS code these days. Sass's ability to create variables, nesting code, and developing mixins which makes it an essential tool in my workflow.

Tools: CSS3 and Sass/Compass

JavaScript

I know the concepts of object oriented programming and Asynchronous Module Definition. I also put comments on JavaScript code for other developers. This may sound not popular these days but I don't have any problems using jQuery to type less verbose JavaScript code or applying third-party jQuery plugins to get a job done in a timely manner. Lastly, JSON is my preferred method to store data objects.

Tools: JavaScript, jQuery, Handlebars.js, Express/Hogan, Require.js, and JSON

Design

I'm a graphic design geek, too. Since my high school days, Adobe Photoshop is my preferred raster graphics application to slice and dice graphics from compositions created by art directors or myself. The popular web image formats JPG, PNG, GIF, and SVG have their own strengths, weaknesses, and purposes. I meticulously pick certain image formats in order to make the web page have a sharp look yet fast loading.

Tools: Adobe Photoshop and Illustrator

Dev Workflow

My front-end web development work environment is a Mac laptop with Node.js installed. The Node.js-based application Grunt task runner handles my automated tasks like minified files. Alongside with Grunt, I use Bower to handle the web front-end libraries such as jQuery or Bootstrap.

Tools: Grunt, Bower, Unix CLI, PHP, and MySQL

Teamwork

I have worked on my projects that required multiple developers. I previously worked with SVN and Microsoft's Team Foundation Server for source control. However, I use Git as my main source control because the ubiquity and the easy flexibility to fork a project to start a new branch. I do have a GitHub account but it is mostly used to track open source projects updates and use as a webhost for this page.

Tools: Git, SVN, Team Foundation Server, Bugzilla, JIRA, and Slack
